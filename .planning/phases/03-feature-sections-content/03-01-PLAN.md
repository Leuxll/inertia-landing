Create `src/components/ui/feature-section.tsx`:
- Client component ("use client" — uses motion for staggered animations)
- Props interface `FeatureSectionProps`:
  - `headline` (string) — philosophical headline text
  - `headlineAccent?` (string) — italic/accent portion of headline (rendered with Heading accent prop)
  - `description` (string) — one sentence max
  - `screenshotSrc` (string) — path to screenshot image
  - `screenshotAlt` (string) — alt text for screenshot
  - `imagePosition?` ("left" | "right", default "left") — which side the phone frame appears on
  - `id?` (string) — section id for nav linking
  - `className?` (string)
- Layout: Uses Section (fullHeight={false}, generous vertical padding `py-24 md:py-32 lg:py-40`) and Container (default size)
- Inner layout: Two-column on desktop (`lg:grid lg:grid-cols-2 lg:gap-16 lg:items-center`), stacked on mobile (`flex flex-col gap-12`)
- When `imagePosition="left"`: PhoneFrame on left, text on right (desktop). On mobile, text always comes first for readability.
- When `imagePosition="right"`: PhoneFrame on right, text on left (desktop). Use `lg:order-2` on the phone frame column to swap without reordering DOM.
- Text column: Heading as h2 with the headline text (if headlineAccent provided, render as two-part: regular text + accent span). Below: Text variant="muted" with description. Stack with `gap-6`.
- Phone column: Centered PhoneFrame with the screenshot.
- Wrap the entire inner content in ScrollReveal variant="stagger" for the container, and each column's content as a motion.div child with fadeUp variants so text and image stagger in.
- For directional flair on desktop: the phone frame column uses slideFromLeft when imagePosition="left" and slideFromRight when imagePosition="right". Import these from animations.ts. On mobile, just use fadeUp for both (simpler). Detect with a CSS approach: apply the directional variant via className on lg: breakpoint, or simply use fadeUp universally and let the stagger timing create visual interest (Claude's discretion on which approach feels better — keep it simple).
